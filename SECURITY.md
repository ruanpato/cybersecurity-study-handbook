# Security Policy · Política de Segurança

🌐 **English** · **Português** (below / abaixo)

> This repository is an **educational handbook and study roadmap** (Markdown content), not a
> running service or application. There is no deployed system to attack. This policy therefore
> covers two things: how to report problems with the **material**, and the ethics rules that
> govern every example in it.

## English

### What "security" means here

The content is **defensive and educational**. By design it contains **no functional malware,
payloads, operationalizable evasion techniques, exploits, or credential-theft tooling**. See
[Chapter 0 — Ethics, legality, and authorization](en-US/HANDBOOK.md#chapter-0).

### Reporting a problem

If you find any of the following, please report it privately **before** opening a public issue:

- content that could be operationalized for harm (e.g., a working exploit/payload that slipped in);
- a leaked secret, credential, or personal datum committed by mistake;
- a malicious link or a dependency/script that does something unexpected.

**How to report:** open a [GitHub Security Advisory](https://github.com/ruanpato/cybersecurity-study-handbook/security/advisories/new) (private), or
contact the maintainer through the repository profile. Please include the file path, lines, and a
short description. Do **not** post exploit details publicly until the issue is resolved.

We follow **coordinated disclosure** (see HANDBOOK §0.4): private report → fix → public note.
A reasonable response target is **30 days** for a content repository.

### Out of scope

Reports about third-party sites linked from [REFERENCES.md](en-US/REFERENCES.md) should go to
those projects, not here.

## Português

### O que "segurança" significa aqui

O conteúdo é **defensivo e educacional**. Por concepção, **não contém malware funcional,
payloads, técnicas de evasão operacionalizáveis, exploits ou ferramentas de roubo de
credenciais**. Ver [Capítulo 0 — Ética, legalidade e autorização](pt-BR/HANDBOOK.md#capitulo-0).

### Como relatar um problema

Se você encontrar qualquer um dos itens abaixo, relate de forma **privada** antes de abrir uma
issue pública:

- conteúdo que possa ser operacionalizado para causar dano (ex.: um exploit/payload funcional que
  tenha escapado);
- um segredo, credencial ou dado pessoal versionado por engano;
- um link malicioso ou um script/dependência com comportamento inesperado.

**Como relatar:** abra um [GitHub Security Advisory](https://github.com/ruanpato/cybersecurity-study-handbook/security/advisories/new) (privado) ou
contate o mantenedor pelo perfil do repositório. Inclua caminho do arquivo, linhas e uma breve
descrição. **Não** publique detalhes de exploração antes da correção.

Seguimos **divulgação coordenada** (ver HANDBOOK §0.4): relato privado → correção → nota pública.
Alvo razoável de resposta: **30 dias** para um repositório de conteúdo.

### Fora de escopo

Problemas em sites de terceiros citados em [REFERENCIAS.md](pt-BR/REFERENCIAS.md) devem ser
relatados àqueles projetos, não aqui.
